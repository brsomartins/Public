Assignment 3

For this assignment, you are asked to use your Student class from assignment 2 and create a GUI which presents the user with dialog windows for inputting information about a student. Once all of the grade information has been entered, your program must display a window which contains a graphics panel on which is drawn the student's letter grade for the course. You will need to read and learn about various GUI and graphics classes provided by Java, and this is intended to give you experience in learning API's. There are also some helpful examples in the lecture slides. (Specifically, the "GUI introduction" slides.)

To show you what is expected, an example solution for this assignment, "Assignment3.jar" has been posted on the course site. To run it from the command line, type: 

java -jar Assignment3.jar
Note that in some newer JDK versions, using some graphics cards with particular settings can cause standard Java UI elements to display incorrectly. If you see partially missing text, disappearing text on buttons, etc., please try running Java with the command line option "-Dsun.java2d.d3d=false", similar to the following: 

java -Dsun.java2d.d3d=false -jar Assignment3.jar

  1. Make a new copy of the files from Assignment 2.


  2. In your student class, create a new function that will return just the letter grade for the course. You can make other modifications if you'd like, but make sure that it is still using Exam and Coursework objects.


  3. Create a new class named Assignment3, which will first use a series of dialogs to prompt the user for all of the necessary information for a student:
    1. First name
    2. Last name
    3. The number of graded activities
    4. If there were n graded activities, prompt the user n times so he/she can enter each activity score
    5. The number of graded homeworks
    6. If there were n graded homeworks, prompt the user n times so he/she can enter each homework score
    7. The score for the midterm
    8. The letter grade for the midterm
    9. The score for the final
    10. The letter grade for the final
    Your assignment should be run using the command "java Assignment3".


  4. When reading in the values for the scores (and number of scores), you should use Integer.parseInt (google it for documentation about how it works) to get the integer value of the value entered by the user. Note that Integer.parseInt throws an exception if given something other than a valid number. If such an exception occurs, you must catch the exception and display a dialog saying "Invalid data entered. Exiting." and then the program should immediately exit.


  5. A Student object should be created and all of that information should be used to calculate the student's letter grade for the course (just as in assignment 2). All aspects of the grade calculation which were required for the previous assignment are required for this assignment, so make sure your code works and correct any bugs that existed in your last assignment. Additionally, recall that in Java, when you want to compare the contents of two String objects, instead of using "==" you should use the "compareTo" method provided by the String class!


  6. Finally, a JFrame object should be created which contains an instance of a panel class that you create (which should be in GradePanel.java and named GradePanel and will extend JPanel similar to the example DrawPanel class in section 4.14 of the book). The panel should display the letter grade for the student. If the letter grade is "A", then draw an "A" using Graphics.drawLine. Make your letter "A" fill up the window, such that it touches the top side and the two bottom corners, even if the window is resized. If the letter grade is "B", "C", "D", or "F", then display the grade using some standard font.

  7. Submit a zip file in the usual manner. The zip file should contain exactly the source files Coursework.java, Exam.java, Student.java, GradePanel.java, Assignment3.java, and a working build script (build.bat or build.bash) which compiles and executes your program. Do not submit any other files.

  8. Here is the rubric the grader will use when running your program. Make sure your code meets each of these requirements. Please keep in mind that this is a very large class and the TA does not have time to fix your code, rename files, etc.:
    1. -90% Code does not compile
    2. -20% Incorrect files contained in the zip file
    3. -20% Build script does not correctly build and run your program
    4. -10% (each) missing dialog box (20% total if it doesn't correctly loop for the required number of activities and/or homeworks)
    5. -5% Incorrect text on dialog box (prompts for wrong information)
    6. -10% If invalid value (i.e. non-integer) is entered, does not display "Invalid data entered. Exiting." then correctly exit
    7. -40% Missing panel with final grade
    8. -30% Final grade is displayed in a dialog rather than drawn on a panel
    9. -20% "A" is not drawn with lines
    10. -10% "A" does not expand or shrink to correctly fill resized window
    11. -10% "B","C", or "D" grade is not written with font
    12. -10% (each) Incorrect final grade
