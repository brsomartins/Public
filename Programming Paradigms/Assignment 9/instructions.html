Assignment 9

  1. For this assignment, you will be creating a basic PHP page which outputs a simple HTML page. You may do initial testing and development on the virtual machine as discussed in the "Instructions for running your own virtual machine for PHP Page" document. However, final development and testing must be on the department's Turing server. See here for instructions on how to set up your web site on Turing. (You may want to make a subdirectory within your main web directory for your test pages.)

  2. Create a file named assignment9.php.

    You can create and edit your page:
    1. On your virtual machine for initial development and testing, and then transfer the file using FileZilla to Turing for final testing. (FileZilla can be found under "Applications" -> "Internet" on the virtual machine after you have executed the command "sudo apt-get install filezilla", and you can use it to connect to Turing using your university login and password.)
    2. Directly on Turing (as described here by using an editor such as `vi')
    3. On your local machine using a basic text editor, and then upload it to Turing by using SFTP or SCP to connect to the server turing.csce.uark.edu with your university login and password
    4. (For Macs) Remotely on Turing but on a drive that you have mapped to your local machine, as described here here for Mac machines.
    From this point, it will be assumed that you are correctly placing your file in the necessary folder on Turing and then accessing it via the appropriate URL for testing.

    Note that the permissions for your file must be set so that everyone can "read" the file. For example, this can be done if you ssh to Turing, "cd" to the correct directory, and run "chmod 600 assignment9.php".

  3. Your page must check to see if the value of a particular global variable named "$_REQUEST" is null and generate the following HTML page if it is:
    • The title is "Assignment 9: no query string"
    • The body contains the following 3 lines at the top: "Hello world!", "firstname lastname's assignment 9.", and "The query string is null."
    • The page should end with a call to the PHP function "phpinfo()"
    and the following page if it is not:
    • The title is "Assignment 9: with a query string"
    • The body contains the following 3 lines at the top: "Hello world!", "firstname lastname's assignment 9.", and "The query string is " + var_dump($_REQUEST)
    • The page should end with a call to the PHP function "phpinfo()"
    You must replace "firstname" with your first name and "lastname" with your last name. For the line that ends with "var_dump($_REQUEST)", you must call the PHP function var_dump() on the $_REQUEST variable and cause the result to be displayed at the end of the line (however is appropriate depending on how you are outputting the line).

  4. An example can be found here:
    • No query string:http://csce.uark.edu/~patitz/assignment9.php
    • With query string:http://csce.uark.edu/~patitz/assignment9.php?test=true

  5. Here is the rubrick the grader will use, so make sure your page meets each of these requirements when run on Turing:

    1. (10 points) Does the PHP file have the correct name?
    2. (15 points) Does the page display content with no visible PHP error (exception) when no query string is used?
    3. (15 points) Does the page display content with no visible PHP error (exception) when a query string is used?
    4. (5 points) Does the page display the lines "hello world" and "firstname lastname's assignment 9" when no query string is used?
    5. (5 points) Does the page display the lines "hello world" and "firstname lastname's assignment 9" when a query string is used?
    6. (20 points) Is the title correctly set based on whether or not a query string is used?
    7. (20 points) Is the correct value (null or var_dump($_REQUEST)) of the query string displayed?
    8. (10 points) Is the PHP environment (i.e. the output of phpinfo()) always displayed?


  6. Submit your file in the usual manner. You should submit only your "assignment9.php" file, it should not be in a compressed folder. Your PHP page must work correctly on Turing for credit, regardless of whether or not it works on another system.
